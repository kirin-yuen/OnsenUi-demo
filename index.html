Managing Pages

Onsen UI can be achieved by defining multiple ons-page components.

A template must be given a unique ID that will be used later on to reference the template
Templates can be defined in two different ways Separate files: *.html files that contain a single ons-page component as the root. Using <template> elements inside index.html
Onsen UI has 3 components that are used to manage multiple pages: ons-navigator, ons-tabbar and ons-splitter.
Navigator If the app contains multiple pages in a hierarchical order, consider using the navigator component. It provides a page stack where you can push and pop pages with transition animations Tab Bar If the app contains multiple pages at the same level, a tab bar is the appropriate choice. Users can pick which page to display by tapping the item on the tab. Splitter Although it is named a splitter, small devices will create a side menu which can be shown/hidden by the user interaction. On larger screens, this will split a single page into two.

These three component are composable, meaning that you can combine them to achieve the desired routing for your specific app.
